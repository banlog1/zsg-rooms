# ZSG Rooms Relay

This Cloudflare Worker routes ZSG Rooms JSON messages through one Durable Object
per room code. Players make outbound secure WebSocket connections, so hosts do
not need port forwarding, Playit, a VPN, or a public IP address.

The relay carries room state only. Minecraft worlds remain local to each player.
See the repository's [technical overview](../docs/TECHNICAL_OVERVIEW.md) for the
message table, snapshot fields, authority model, and privacy boundary.

## Requirements

- Node.js 18 or newer
- A Cloudflare account with Workers enabled
- Wrangler authentication for deployment

Durable Objects use the SQLite backend configured in `wrangler.jsonc`. The
service is intended for small private rooms and can be deployed on Cloudflare's
free Workers allocation, subject to Cloudflare's current limits.

## Install and Run Locally

```powershell
npm.cmd install
npm.cmd run dev
```

Wrangler prints a local HTTP endpoint. The mod accepts HTTP/HTTPS endpoints and
bare hostnames, converting them to WS/WSS for `/room/<room-code>` connections.

## Deploy

```powershell
npx.cmd wrangler login
npm.cmd run deploy
```

Enter the resulting `https://zsg-rooms-relay.<account>.workers.dev` URL, or just
its hostname, in the mod's Relay field. Every participant in a room must use the
same relay deployment.

## Verify a Deployment

```powershell
npm.cmd run smoke -- https://your-relay.workers.dev
```

The smoke test creates a unique room, connects a host and guest, interrupts the
host, verifies the reconnect grace message, reconnects with the same ownership
token, and confirms that the stored room snapshot is restored.

## Connection Rules

- The host authenticates with a random bearer token generated by the mod.
- A second host cannot claim the same stored room with a different token.
- Guests can connect only while the host is online, except when reclaiming an
  existing guest identity during the disconnect grace period.
- Guest message types are limited by `GUEST_ACTIONS` in `src/index.js`.
- Messages larger than 256 KiB are rejected.
- Clean close code 1000 leaves immediately.
- Abnormal host and guest disconnects receive a 60-second grace period.
- A Durable Object alarm finalizes identities that do not reconnect.

The mod sends WebSocket protocol heartbeats and performs its own bounded
reconnect attempts. The Worker stores room ownership, maximum-player state, the
latest room snapshot, and reconnect metadata.

## Commands

```powershell

```
